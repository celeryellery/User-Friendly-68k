Opcode Table


General Information:

  • The default number system when writing 68k assembly is decimal
  • All hexadecimal numbers are preceded by $
  • All binary numbers are preceded by %
  • If you want to program to interpret any number just as a regular numeric value and nothing else, write # in front of it. Then it is called an immediate value.
  • Assembly commands that talk about two locations have the source written first, then the destination.
  • There are many different methods to tell the processor where the source and destination are. These different methods are called addressing modes. Refer to the 68000 Programmer's Reference Manual for more information on these.
  • The notation Dn refers to any data register, D0-D7. Similarly, An refers to any address register A0-A6. (A7 is the stack pointer so it is not included.)
  • Unless otherwise noted, all information on this page come from the 68000 Programmer's Reference Manual, Copyright Motorola Inc.
